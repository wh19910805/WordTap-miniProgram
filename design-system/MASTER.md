## 交互规范

### 触控目标

- **最小尺寸**: 44×44pt (iOS) / 48×48dp (Android)
- **间距**: 最小 8px 间隔

### 动画时长

- **微交互**: 150-300ms
- **页面过渡**: ≤400ms
- **避免**: >500ms 的动画

### 状态反馈

- **按下状态**: 80-150ms 内提供视觉反馈
- **加载状态**: >300ms 显示骨架屏
- **错误反馈**: 清晰的错误信息和恢复路径

## 页面特定规则

### 学习页面 (learning)

- **当前句子高亮**: 使用 `active` 类，边框颜色 `--primary-color`
- **输入框**: 隐藏式输入框，最小触控区域 44×44px
- **底部按钮栏**: 固定定位，z-index: 10

### 设置页面 (settings)

- **设置项**: 使用开关、单选、滑块等原生组件
- **触控目标**: 所有可点击元素最小 44×44px
- **间距**: 使用 4pt 基准系统

## 可访问性检查清单

- [ ] 所有文本对比度 ≥ 4.5:1
- [ ] 所有交互元素 ≥ 44×44px
- [ ] 图标有辅助文字或 aria-label
- [ ] 表单元素有标签
- [ ] 错误信息清晰且有恢复路径
- [ ] 支持动态文字大小
- [ ] 支持减少动画模式

## 反模式避免

- [ ] 不使用 emoji 作为结构图标
- [ ] 不混合使用 filled 和 outline 图标
- [ ] 不使用 hover 作为主要交互方式
- [ ] 不固定像素宽度容器
- [ ] 不禁用缩放
